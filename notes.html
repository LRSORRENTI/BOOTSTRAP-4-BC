So what exactly is Bootstrap?

Well according to the offical docs, Bootstrap helps build responsive, mobile-first projects on the web with the world's most popular front-end component library

It's also sometimes referred to as a CSS framework, but at the end of the day it's a single CSS file, a single JS file that you include in your html documents

These documents are very long, and technically behind the scenes, 50 files or so that are compiled into one.

But what we do is we go into our html markup, and add classes to our elements. The majority of this course is adding classes to html elements

Bootstrap includes dozens and dozens of pre-built components like:
  • Buttons
  • Forms
  • Navbars
  • The super useful GRID system

    • The grid system is very crucial, some consider it to be the most useful thing, some people only use bootstrap for it's grid system

  • Utilities

    • In bootstrap 4 there are these 'utilities' that help us do common operations, they're very useful. Stuff for padding, margin, border, or spacing, all sorts of different things

Why use bootstrap? Pros and cons

This section we'll go over the benefits of using bootstrap, and the next section we'll go over the downsides of bootstrap

  • Fast Development

    • If you're goal is making a nice looking, functional website bootstrap is amazing, but if you wanted to try to build a game playable in the browser bootstrap isn't going to help

  • Browswer Compatibility

    • Bootstrap supports almost all browsers, but the nice thing bootstrap does is coordinate the css based on the browser, not everything is unified across all broswers, but bootstrap will take care of that for you

  • Responsive layouts

    • When you want to make responsive websites, and you don't use bootstrap, it's doable, just takes longer, and a bit more tedious

  • It can be fun!

This section we'll go over the downsides of bootstrap

Do Bootstrap websites all look the same? If you're always using the same nav bars, and forms, aren't all your websites going to look the same?

The answer is that yes they can, but also no, because we can always customize!

The point is that we use bootstrap as a tool, not the only thing we use to build a website. We use it in tandem with our own CSS

Intro to: BOOTSTRAP 4 BREAKPOINTS

One of the benefits of BOOTSTRAP is that it helps create responsive designs very quickly, making fully responsive websites in vanilla CSS takes longer, and sometimes is the right choice, but sometimes using libraries like BOOTSTRAP or TAILWIND CSS can be the correct tools depending on the job

When we shrink a window that has a responsive design, there's a certain point when you mininmize the window where the content of the page will responsively change to fit the size of the window. That exact point where the window responsively changes is called the BREAKPOINT

We could set our own breakpoints with speciific pixel sizes, like at 1102px wide chnage the webpage resposively, then if it shrinks to 700px wide, change it again

BOOTSTRAP has some built-in numbers that it works with, for ex: on the BOOTSTRAP documentation, under navbars we can see the underlying code that will make that navbar resposive: nav class="navbar navbar-expand-lg navbar-light bg-light" >

In the above code snippet, it's the 'navbar-expand-lg' that will cause the webpage to responsively change, here's the official BOOTSTRAP docs on RESPONSIVE BREAKPOINTS:

Since Bootstrap is developed to be mobile first, we use a handful of media queries to create sensible breakpoints for our layouts and interfaces. These breakpoints are mostly based on minimum viewport widths and allow us to scale up elements as the viewport changes. Bootstrap primarily uses the following media query ranges—or breakpoints—in our source Sass files for our layout, grid system, and components.

// Extra small devices (portrait phones, less than 576px) // No media query for `xs` since this is the default in Bootstrap // Small devices (landscape phones, 576px and up) @media (min-width: 576px) { ... } // Medium devices (tablets, 768px and up) @media (min-width: 768px) { ... } // Large devices (desktops, 992px and up) @media (min-width: 992px) { ... } // Extra large devices (large desktops, 1200px and up) @media (min-width: 1200px) { ... } -->

QUICK BREAKPOINT LIST:

  • SMALL(sm)(phones): portrait = < 576px, landscape = 576 - 700ishpx
  • MEDIUM(md)(tablets): > 768px
  • LARGE(lg): 992px
  • XTRA-LARGE(XL): 1200px

This is the responsive design example colt used in his BREAKPOINTS demonstration:

Inside our demo app.css: body{ background-color: lightsalmon ; } @media (min-width: 576px) { body{ background-color: lightblue; } } @media (min-width: 768px) { body{ background-color: lightgreen; } } @media (min-width: 992px) { body{ background-color: lightgoldenrodyellow; } } @media (min-width: 1200px) { body{ background-color: lightgrey; } }

Essentially this shows that when altering screen-size, starting with the smallest which is salmon, as you expand the screen the colors change based on each specified pixel size, once the window reaches 577px, it chnages to lightblue, then when it reaches 768, it changes to lightgreen, and so on until it reaches the max value with is lightgrey

In summary, these are underlying pixel values that bootstrap uses to resposively change the layout of a webpage, you don't need to memorize the pixel values

SPACING UTILITIES: Margin and padding

Next up is spacing utilities, they're very useful, they come up a lot! They're used very frequently

From the BOOTSTRAP docs:

Spacing: Bootstrap includes a wide range of shorthand responsive margin and padding utility classes to modify an element's appearance.

How it works Assign responsive-friendly margin or padding values to an element or a subset of its sides with shorthand classes. Includes support for individual properties, all properties, and vertical and horizontal properties. Classes are built from a default Sass map ranging from .25rem to 3rem.

Notation Spacing utilities that apply to all breakpoints, from xs to xl, have no breakpoint abbreviation in them. This is because those classes are applied from min-width: 0 and up, and thus are not bound by a media query. The remaining breakpoints, however, do include a breakpoint abbreviation. The classes are named using the format {property}{sides}-{size} for xs and {property}{sides}-{breakpoint}-{size} for sm, md, lg, and xl.

Where property is one of: m - for classes that set margin p - for classes that set padding Where sides is one of: t - for classes that set margin-top or padding-top b - for classes that set margin-bottom or padding-bottom l - for classes that set margin-left or padding-left r - for classes that set margin-right or padding-right x - for classes that set both *-left and *-right y - for classes that set both *-top and *-bottom blank - for classes that set a margin or padding on all 4 sides of the element Where size is one of: 0 - for classes that eliminate the margin or padding by setting it to 0 1 - (by default) for classes that set the margin or padding to $spacer * .25 2 - (by default) for classes that set the margin or padding to $spacer * .5 3 - (by default) for classes that set the margin or padding to $spacer 4 - (by default) for classes that set the margin or padding to $spacer * 1.5 5 - (by default) for classes that set the margin or padding to $spacer * 3 auto - for classes that set the margin to auto (You can add more sizes by adding entries to the $spacers Sass map variable.)

Examples Here are some representative examples of these classes: .mt-0 { margin-top: 0 !important; } .ml-1 { margin-left: ($spacer * .25) !important; } .px-2 { padding-left: ($spacer * .5) !important; padding-right: ($spacer * .5) !important; } .p-3 { padding: $spacer !important; } Horizontal centering Additionally, Bootstrap also includes an .mx-auto class for horizontally centering fixed-width block level content—that is, content that has display: block and a width set—by setting the horizontal margins to auto. INTRO TO THE GRID SYSTEM

The GRID system is possibly the most commonly used tool in BOOTSTRAP, but on it's own there's nothing to see, it's not a component like a navbar or a button, it's more of an underlying structure behind the scenes, that helps layout webpages

The grid system is responsive, so at certain sizes the available cards/content will shift to match the specified breakpoints

AN IMPORTANT THING TO NOTE IS THAT 'CONTAINERS' ARE REQUIRED WHEN USING THE GRID SYSTEM

If we want our content to go horizontally across our container, we use row, then within that row, we divide it using column, column classes 'col-sm' or 'col-md'
